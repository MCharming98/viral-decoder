# HTML Report

Turn a ViralDecoder markdown report into a **self-contained infographic HTML page** — one file, no build step, opens in any browser.

## Input

Prefer `reports/{username}/complete-report.md`. Fall back to merging individual reports in the same directory:

- `profile.md`, `viral-playbook.md`, `activity.md`, `engagement.md`, `content-insights.md`

When chart data is missing from markdown, recompute from `load_stored_tweets(username)` and `get_user_by_username`.

## Output

- **Path:** `reports/{username}/complete-report.html`
- **Format:** single `.html` file with inline CSS and Chart.js via CDN
- Create `reports/{username}/` if needed

## Workflow

1. Read the markdown report(s) for `{username}`.
2. Extract metrics, summaries, top posts, themes, and sample metadata.
3. If needed, load tweets and profile for chart-ready datasets.
4. Build the HTML page following the layout and chart map below.
5. Save to `reports/{username}/complete-report.html`.
6. Open in browser to verify layout and that all charts render.

## Design principles

- Infographic style
- Design clean, structured visual layout
- Use visualization charts for metrics, e.g pie chart, heatmap

### Visual system

| Token | Value |
|-------|-------|
| Background | `#0f1117` page, `#1a1d27` cards |
| Text | `#e8eaed` primary, `#9aa0a6` secondary |
| Accent | `#1d9bf0` (X blue), `#00ba7c` positive, `#f4212e` alert |
| Font | `system-ui, -apple-system, "Segoe UI", sans-serif` |
| Radius | `12px` cards, `8px` chips |
| Spacing | `24px` section gap, `16px` card padding |

### Layout structure

Three major sections. Charts live only in section 3.

```
┌─────────────────────────────────────────────┐
│ 1. PROFILE                                  │
│    HERO: avatar, name, @handle, bio, chips  │
│    Stat cards: followers, following, posts  │
│    Profile summary (2-3 sentences)          │
├─────────────────────────────────────────────┤
│ 2. VIRAL PLAYBOOK                           │
│    Core formula, SOP steps, hook library    │
│    Post anatomy, content arc, anti-patterns │
├─────────────────────────────────────────────┤
│ 3. DETAILED STATISTICS                      │
│    Reach bar │ Content mix pie               │
│    Language  │ Media vs text engagement     │
│    Heatmap   │ Engagement averages          │
│    Top themes (horizontal bars)             │
│    Top posts (ranked cards)                 │
│    Assessment + key takeaways               │
├─────────────────────────────────────────────┤
│ FOOTER: generated by ViralDecoder, date range  │
└─────────────────────────────────────────────┘
```

- Max content width: `1100px`, centered
- CSS Grid for dashboard panels; stack to single column below `768px`
- Section headers with a thin accent left border
- All external links open in new tab (`target="_blank" rel="noopener"`)

## Required charts

Map report metrics to these chart types. Skip a chart only when data is genuinely unavailable; note the gap in the UI.

| Chart | Type | Data source | Section |
|-------|------|-------------|---------|
| Content mix | **Pie / donut** | content-insights type shares | Activity / Content |
| Language breakdown | **Pie** | activity `lang` counts | Activity |
| Media mix | **Pie** | photo / video / GIF / none | Activity |
| Posting heatmap | **Heatmap** | day-of-week (rows) × hour UTC (cols), post counts | Activity |
| Reach metrics | **Bar** | followers, following, posts, listed | Profile |
| Engagement averages | **Grouped bar** | likes, RTs, replies, quotes, impressions (mean) | Engagement |
| Theme performance | **Horizontal bar** | top themes with avg engagement | Content insights |
| Media vs text | **Bar** | avg engagement split | Engagement |

### Heatmap implementation

Use Chart.js `matrix` plugin or a CSS grid heatmap (preferred — no extra plugin):

```html
<div class="heatmap">
  <!-- 7 rows (Mon–Sun) × 24 cols; cell opacity = count / max -->
</div>
```

Label rows with day names and columns with every 4th hour (0, 4, 8, 12, 16, 20).

### Chart.js setup

Load from CDN in `<head>`:

```html
<script src="https://cdn.jsdelivr.net/npm/chart.js@4"></script>
```

- Use `responsive: true`, `maintainAspectRatio: false`
- Dark-theme chart defaults: light grid lines (`rgba(255,255,255,0.08)`), legend labels `#9aa0a6`
- Wrap each canvas in a fixed-height container (`280px` standard, `360px` for heatmap)
- Assign distinct palette: `['#1d9bf0','#00ba7c','#f91880','#ffad1f','#794bc4','#17bf63']`

## Content mapping

Pull these from the markdown sections:

| Markdown section | HTML treatment |
|------------------|----------------|
| **1. Profile** | Hero card + stat row + summary block |
| Profile overview / reach / signals | Hero, stat cards, chips |
| **2. Viral playbook** | Expandable playbook panel (SOP, hooks, arc) |
| **3. Detailed statistics** | All charts + top posts + assessment |
| Activity cadence | Heatmap + cadence callouts |
| Engagement averages | Grouped bar chart |
| Top posts | Ranked list cards (#1–#5) with links to `x.com` |
| Content mix table | Donut chart + legend |
| Top themes | Horizontal bar chart |
| Assessment / key takeaways | Bulleted panel + pull-quote |

Preserve all tweet links as clickable anchors. Never strip permalinks.

## HTML skeleton

Start from [reference.html](reference.html). Replace placeholders with real data. Keep the file self-contained except Chart.js CDN.

Key elements to include:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>ViralDecoder — @{username}</title>

<body>
  <main class="report">
    <header class="hero">...</header>
    <section class="stats-row">...</section>
    <section class="charts-grid">...</section>
    <section class="takeaways">...</section>
    <footer class="meta">Generated by ViralDecoder · {date range}</footer>

  <script src="https://cdn.jsdelivr.net/npm/chart.js@4"></script>
  <script>/* inline chart init with real data */</script>

</html>
```

## Quality checklist

- [ ] Infographic style — visual hierarchy, not a markdown dump
- [ ] Clean, structured visual layout — grid, consistent spacing, readable typography
- [ ] At least one pie chart, one bar chart, and one heatmap present
- [ ] Hero section with profile image (from `profile_image_url` if available)
- [ ] All cited posts link to `https://x.com/...`
- [ ] Sample limitations noted in footer (date range, post count)
- [ ] Mobile-responsive (single column below 768px)
- [ ] No broken chart containers or empty sections without explanation

## Notes

- Do not add npm, webpack, or external CSS frameworks — keep it one HTML file.
- Profile image: use the URL from profile data; fall back to initials avatar.
- Format large numbers with locale separators (e.g. `1,696`).
- Engagement rate: show as percentage with one decimal.
- If only `profile.md` exists (no tweet data), render profile section and stat cards; omit activity/engagement charts and note limited data.
