# Full Report

Produce a comprehensive report by combining all ViralDecoder report references.

See [SKILL.md](../SKILL.md) for shared tools and setup.

## Tools

| Tool | Module | Purpose |
|------|--------|---------|
| `get_user_by_username` | `scripts/profile.py` | Profile lookup |
| `get_and_store_user_tweets` | `scripts/timeline.py` | Fetch, dedupe, and store timeline |
| `load_stored_tweets` | `scripts/store.py` | Read full saved dataset for analysis |

## Workflow

1. Resolve the user with `get_user_by_username`.
2. Fetch timeline data with `get_and_store_user_tweets` (paginate until covered).
3. Load `load_stored_tweets(username)` as the analysis dataset.
4. Follow [profile-report](profile-report.md) for account metadata and reach.
5. Follow [activity-report](activity-report.md) for posting patterns.
6. Follow [engagement-report](engagement-report.md) for performance metrics.
7. Follow [content-insights](content-insights.md) for audience, content strategy, and voice analysis.
8. Follow [viral-playbook](viral-playbook.md) for the synthesized SOP section.
9. Merge into the combined template below.
10. Save the report to `reports/{username}/complete-report.md`.

### Data collection

```python
user = get_user_by_username(client, username)
payload = get_and_store_user_tweets(
    client,
    user["id"],
    username=user["username"],
    start_time=start_time,   # optional
    end_time=end_time,
)

while payload.get("meta", {}).get("next_token"):
    payload = get_and_store_user_tweets(
        client,
        user["id"],
        username=user["username"],
        start_time=start_time,
        end_time=end_time,
        pagination_token=payload["meta"]["next_token"],
    )

tweets = load_stored_tweets(user["username"])
```

## Linking

All sub-reports must link to X when referencing the account or a specific post:

- **Profile:** `[@{username}](https://x.com/{username})` on first mention in the report title and profile section.
- **Any tweet cited:** use the stored `url` field — e.g. `[truncated text…]({url})`.
- **Fallback:** if `url` is missing, use `https://x.com/{username}/status/{id}`.

Never mention a tweet anywhere in the report without its link.

## Combined template

The report has **three major sections**. Sub-reports (activity, engagement, content-insights) merge into section 3.

```markdown
# ViralDecoder: [@{username}](https://x.com/{username})

> Generated by ViralDecoder

---

## 1. Profile
{Output from profile-report — overview, reach, signals, summary only; keep high-level}

---

## 2. {username}'s viral playbook
{Output from viral-playbook — omit the top-level title and sample block; keep from Core formula through Bottom line}

---

## 3. Detailed statistics

> Sample: {N} posts · {date range} · {data source}  
> Charts: see complete-report.html

### Activity
{Output from activity-report — cadence and content tables, patterns}

### Engagement
{Output from engagement-report — averages, top/underperformers}

### Content insights
{Output from content-insights — audience, mix, themes, voice}

### Assessment
{3-5 sentences synthesizing identity, behavior, and influence}

**Key takeaways**
- {bullet 1}
- {bullet 2}
- {bullet 3}
```

## Save report

- **Path:** `reports/{username}/complete-report.md` (strip leading `@` from username)
- **Format:** markdown file containing the combined template above
- Create `reports/{username}/` if it does not exist
- When running sub-reports individually, each saves to its own file in the same directory:
  - `reports/{username}/profile.md`
  - `reports/{username}/activity.md`
  - `reports/{username}/engagement.md`
  - `reports/{username}/content-insights.md`
  - `reports/{username}/viral-playbook.md`

## Notes

- `get_and_store_user_tweets` handles fetch, deduplication, and storage — do not call `store_tweets` separately.
- When the requested time window is fully cached, it skips the API and returns stored data.
- Use `load_stored_tweets` for all tweet analysis, not individual fetch pages.
- Each stored tweet includes `url` (permalink), `media` metadata, and `entities.urls` for content analysis.
- Keep each section concise; the synthesis matters most.
- Flag data limitations (protected account, small sample, missing impressions, cached-only data).
- Stay factual — this is analysis, not judgment.
- When citing any tweet, always include its permalink as a markdown link.
